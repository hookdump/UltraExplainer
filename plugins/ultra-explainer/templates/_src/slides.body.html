Engineering review · Q3 2026

Edge Gateway Migration

From a single Node proxy to edge-native routing

01

The problem

Why one proxy process can't carry the traffic anymore.

Before

Monolith proxy

  • All traffic through one Node process
  • Per-instance rate limits, not distributed
  • Synchronous JWT adds ~40ms
  • No circuit breaking — failures cascade

After

Edge-native

  • Workers at 340 edge locations
  • Durable Objects for shared state
  • Async JWT with key caching
  • Automatic per-upstream breakers
