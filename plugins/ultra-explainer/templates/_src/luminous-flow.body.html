Cache hit · the common case

Cache answers directly. ~4 ms, zero database load.

The request stops at the cache. The database never sees it — which is the entire point of putting a cache there. For a read-heavy service this is 90–99% of traffic.

Cache miss · the tail

Cache forwards to the origin, fills, then answers. ~92 ms this time.

The miss pays the full origin round-trip and writes the result back, so the next identical read is a hit. A cold cache is all misses — which is why warm-up traffic looks nothing like steady state.

What the toggle is and isn't. Both paths exist in the diagram at all times — the control only changes which one is lit. Hover any node to isolate just its connections. Latencies here are illustrative; measure your own cache and origin before quoting them.
