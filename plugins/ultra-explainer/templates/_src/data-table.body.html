Gap detail — the unfired event
Gap: stoq:restock-modal:submitted. The request explicitly asks to fire this on form submission. The plan uses Stoq's openInlineForm inside a custom modal but doesn't say whether Stoq dispatches the event in that context. If Klaviyo, analytics, or theme JS listen for it, missing it silently breaks the submission pipeline. Recommend adding an explicit dispatchEvent as a safety net. (Inferred from the plan text; not confirmed against Stoq's runtime.)
